# 后端帖子系统实现指南

## 📋 步骤说明

### 1. 运行代码生成（Makefile）

首先需要运行 go-zero 的代码生成工具来生成 handler 和 logic 的骨架代码：

```bash
cd backend
make gen-api
```

这会根据 `super.api` 文件自动生成：
- `backend/api/internal/handler/post/` - Post相关的handler
- `backend/api/internal/handler/comment/` - Comment相关的handler
- `backend/api/internal/logic/post/` - Post相关的logic
- `backend/api/internal/logic/comment/` - Comment相关的logic
- `backend/api/internal/types/` - 类型定义

### 2. 更新数据库模型

需要在数据库初始化时添加Post和Comment表。检查 `backend/utils/db.go` 文件，确保包含：

```go
db.AutoMigrate(
    &model.User{},
    &model.VipPlan{},
    &model.VipOrder{},
    &model.VipRecord{},
    &model.Post{},        // 新增
    &model.PostLike{},    // 新增
    &model.Comment{},     // 新增
    &model.CommentLike{}, // 新增
)
```

### 3. 实现RPC服务（可选）

如果使用RPC架构，还需要：
1. 在 `backend/rpc/super.proto` 中添加Post和Comment相关的消息定义
2. 在 `backend` 下运行 `make gen-rpc` 生成 RPC 代码（命令见 `Makefile`）
3. 实现RPC logic

### 4. 实现业务逻辑

参考现有的logic实现，完成Post和Comment的业务逻辑。

## ⚠️ 注意事项

1. **ID类型转换**: 数据库使用uint，API使用string，需要转换
2. **图片存储**: Post的Images字段存储JSON格式的字符串数组
3. **点赞逻辑**: 需要检查用户是否已点赞，避免重复点赞
4. **评论计数**: 创建/删除评论时需要更新Post的Comments字段
5. **点赞计数**: 点赞/取消点赞时需要更新Post/Comment的Likes字段
