# 应用配置与全局常量分层约定

本文档用于统一 `AppConfig` 相关常量的归档位置与扩展方式，降低“字符串键散落”和“默认值重复定义”带来的维护成本。

## 分层规则

- `lib/config/**`
  - 放置配置读写逻辑（如：存储、校验、导入导出、组合查询）。
  - 负责“行为”，不再内联定义大量键名字符串。
- `lib/constants/**`
  - 放置可复用的常量定义（如：存储键、默认值、序列化字段名）。
  - 负责“数据定义”，不承担业务逻辑。

## AppConfig 常量归档

`lib/constants/app_config_constants.dart` 统一维护以下三类常量：

- `AppConfigStorageKeys`
  - Flutter Secure Storage 的 key（例如 `api_url`、`model_name`）。
- `AppConfigDefaults`
  - 配置默认值（例如默认 API 地址、默认步骤超时）。
- `AppConfigFields`
  - 配置 Map 导入导出的字段名（例如 `apiUrl`、`enableLogging`）。

## 新增配置项 checklist

新增一个 `AppConfig` 字段时，按顺序执行：

1. 在 `AppConfigStorageKeys` 中添加存储 key。
2. 若需要默认值，在 `AppConfigDefaults` 中补充默认值。
3. 若该字段参与导入导出，在 `AppConfigFields` 中添加字段名。
4. 在 `lib/config/app_config.dart` 中补充对应 get/set 与导入导出逻辑。

## 命名约定

- Storage key：使用 `snake_case`（与现有持久化数据兼容）。
- Config field：使用 `camelCase`（与 Dart Map 字段风格一致）。
- 默认值命名：与字段语义一致，避免缩写与歧义。
