# 响应式页面模板与检查清单

目标：新页面在不同尺寸、不同字体缩放下都不出现 `OVERFLOWED`。

## 1) 默认页面骨架（必须）

新页面优先使用：

- `lib/widgets/layout/adaptive_page_scaffold.dart`
- `lib/widgets/layout/adaptive_section_card.dart`
- `lib/utils/responsive.dart`

示例：

```dart
return AdaptivePageScaffold(
  appBar: AppBar(title: const Text('示例')),
  child: Column(
    crossAxisAlignment: CrossAxisAlignment.stretch,
    children: const [
      AdaptiveSectionCard(
        child: Text('内容区块'),
      ),
    ],
  ),
);
```

## 2) 高风险写法（禁止）

- 大块内容写死固定高度（如 `height: 180`）但内部又是 `Column/Row + 文本`
- `Row` 里直接放长文本，不加 `Expanded/Flexible`
- `Text` 不设 `maxLines + overflow`
- 弹窗内容直接用 `Column` 不滚动

## 3) 标准兜底策略（必须）

- `Row` 文本：`Expanded/Flexible + maxLines + TextOverflow.ellipsis`
- 卡片区：优先使用 `AdaptiveSectionCard`
- 弹窗区：统一包裹 `AdaptiveDialogContent`
- 页面区：优先 `AdaptivePageScaffold(scrollable: true)`，避免极端尺寸溢出

## 4) 提测尺寸（必须）

每个新页面至少验证：

- `323 x 675`
- `400 x 675`
- `467 x 675`
- 系统字体缩放：`1.2` / `1.3`

## 5) 自查标准

- 页面无黄色黑纹 `OVERFLOWED` 条
- 顶部/底部安全区无裁切
- 按钮点击区域可用、文案可读
- 不因“2 字/3 字标签差异”触发布局崩溃
