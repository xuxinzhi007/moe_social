# 登录 / 注册 / Feed 冒烟测试清单

## 测试目标

- 验证本次性能优化后功能正确、交互正常、无明显回归。
- 覆盖登录、注册与 Feed 相关页面（首页、话题页、用户主页）关键链路。

## 一、登录页（`/login`）

- 邮箱输入后缀候选是否正常弹出/收起（含快速输入与删除）。
- 点击候选后缀是否正确回填，并自动将焦点切换到密码框。
- 候选气泡在失焦时是否稳定收起（无闪烁、无误触）。
- 错误账号/密码时是否提示友好错误文案。
- 登录成功后是否进入首页，通知初始化与页面跳转是否正常。

### 边界用例

- 输入仅空格、仅 `@`、多个 `@`、超长邮箱。
- 输入 10 位 Moe 号登录是否通过校验。
- 连续快速点击登录按钮是否被加载态正确防重入。

## 二、注册页（`/register`）

- 用户名、邮箱、密码、确认密码校验是否正确。
- 邮箱候选气泡是否与登录页一致（弹出、回填、收起时序）。
- 注册成功后 Moe 号弹窗是否正常展示，关闭后是否正确返回登录页。
- 注册失败时错误提示是否符合预期（无崩溃、无卡死）。

### 边界用例

- 密码与确认密码不一致提示。
- 邮箱尾部空格自动 trim 后是否可正常注册。
- 网络慢/失败场景下是否可重试，按钮状态是否恢复。

## 三、首页（`/home` feed）

- 首屏加载、下拉刷新、上拉加载更多是否正常。
- 点赞后心形与计数是否即时更新（不需要整页闪动）。
- 连续点赞/取消点赞是否状态稳定、数字不乱跳。
- 切换热门/最新/关注 tab 后列表数据是否正确切换。
- 话题标签筛选（含“全部”）是否准确过滤和恢复。

### 交互一致性

- 话题 chip 点击、加载更多按钮、stories 区域点击是否有触摸反馈（水波纹/点击态）。

### 错误态

- 首屏加载失败：错误态卡片与重试按钮是否可用。
- 加载更多失败：底部错误提示与重试按钮是否可用。

## 四、话题动态页（`/topic-posts`）

- 进入话题页后首屏加载是否正确显示。
- 列表中点赞后是否即时刷新心形与计数（不出现整页闪动）。
- 连续点赞/取消点赞后，返回首页再回到话题页状态是否一致。
- 上拉加载更多后继续点赞，状态是否正确且无明显卡顿。
- 删除作者自己的动态后，列表和点赞状态是否同步清理。

## 五、用户主页（`/user-profile`）

- 用户动态列表中点赞是否即时更新且无整页抖动。
- 点赞后进入动态详情再返回，点赞状态是否一致。
- 编辑动态后点赞/评论数是否保持正确快照。
- 删除动态后列表是否正确移除，且无残留点赞状态。

## 六、性能观察（手工）

- 在首页连续滚动 3~5 分钟，观察是否出现明显掉帧。
- 在滚动中持续点赞，观察是否出现整页重绘导致卡顿。
- 反复切换 tab + 刷新，观察内存是否持续异常上涨。

## 七、内存与状态回收观察点

- 多刷帖子后，点赞状态仍能正确显示（无随机回退）。
- 长时间使用后（10 分钟+），应用是否出现明显内存膨胀。
- 从首页 -> 话题页 -> 用户主页来回切换后，点赞状态是否保持一致。
- 删除帖子后，重复进入相关页面，不应出现旧点赞计数“幽灵回显”。

## 八、建议测试设备组合

- Android 中端机（重点看掉帧与输入手感）。
- iOS 真机（重点看滚动/动画流畅度与弹窗时序）。
- 网络环境：Wi-Fi / 4G / 弱网（高延迟）各一轮。

## 九、回归通过标准（建议）

- 无崩溃、无阻断主流程的功能错误。
- 登录/注册/首页/话题页/用户主页主链路可完整跑通。
- 点赞与邮箱候选交互无明显卡顿或错乱。
- 弱网失败可恢复，可重复操作不异常。
