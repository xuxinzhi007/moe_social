# Ollama 用户级记忆缺陷与演进方案

## 1. 背景与目标

当前项目已接入 Ollama，并具备基础对话、模型管理、记忆提取与画像整理能力。  
但整体实现仍偏“可用 Demo”，距离“懂用户、会学习、可持续训练”的产品目标有明显差距。

目标定义：

- 实现 **用户级长期记忆**：能稳定记住用户偏好、习惯、重要事实和长期计划。
- 实现 **持续学习**：对每轮对话进行结构化吸收，自动更新记忆而非简单堆积。
- 实现 **可训练能力**：支持把用户反馈与高质量样本沉淀为可复用训练/微调数据。
- 实现 **可控可审计**：记忆可查看、可编辑、可删除，可追溯来源与更新时间。

---

## 2. 当前缺陷清单（按优先级）

## P0（高优先级，影响安全/稳定）

- LLM 管理接口鉴权不足（下载模型、删除模型、创建模型、聊天等存在被滥用风险）。
- “停止生成”仅停前端状态，未真正中断后端推理，请求仍可能继续消耗资源。
- 流式交互名义上启用 `stream`，但前端仍是整包等待，无法实现真实增量输出。

## P1（中优先级，影响效果与体验）

- 记忆提取并发治理不足：后台异步提取缺少队列/限流，压力场景下易堆积。
- 记忆日志存在敏感信息泄露风险：原始提取结果直接打印，不利于隐私合规。
- 记忆配置存在“名义字段未生效”问题（部分参数未真正参与调度触发）。
- 筛选与排序等辅助功能逻辑不完整，影响智能体管理体验。

## P2（低优先级，影响可维护性）

- 会话与使用统计部分仍为本地临时态，跨设备与重启后连续性较弱。
- 局部页面状态更新粒度较粗，长会话下渲染性能有优化空间。
- 部分空安全和异常处理路径仍可进一步收敛。

---

## 3. 目标能力拆解（你要的“懂用户 AI”）

为实现“记住用户爱好、理解操作/对话习惯、持续学习并支持训练”，建议按四层能力建设：

1. **记忆采集层**  
   - 对每轮对话提取结构化事实（偏好、禁忌、计划、关系、风格）。
   - 支持冲突检测（新旧事实矛盾时更新而非叠加）。

2. **记忆治理层**  
   - 去重、合并、过期、可信度评分、来源追踪（来源会话+消息 ID）。
   - 用户可视化管理：查看/修订/删除/冻结某条记忆。

3. **推理注入层**  
   - 根据场景注入最相关记忆（而非全量注入），控制 token 成本与噪声。
   - 注入策略可配：仅画像、画像+高优先级事实、任务型临时记忆等。

4. **训练闭环层**  
   - 把用户正反馈（点赞、采纳、复制）和人工修订沉淀为偏好样本。
   - 形成监督数据集（SFT/DPO 候选），支持周期化离线训练或蒸馏。

---

## 4. 推荐演进路线图

## Phase 1：安全与稳定基线（1 周）

- 为 `/api/llm/*` 与 `/api/llm/*/raw` 增加鉴权与限流。
- 打通真正的可取消链路（前端取消 -> 后端 context cancel -> Ollama 中断）。
- 改造成真实流式输出（SSE/NDJSON 增量渲染）。
- 统一错误码与超时重试策略，避免前后端语义不一致。

验收标准：

- 未授权无法操作模型管理接口。
- 停止生成后，后端请求能在秒级中止。
- 首字延迟与整段返回差异可观测。

## Phase 2：用户级记忆 MVP（1-2 周）

- 记忆数据模型升级：`user_id + memory_type + confidence + source + updated_at`。
- 建立冲突更新规则（例如偏好反转、计划变更、关系变更）。
- 记忆提取任务队列化（串行/限并发），降低高峰期不稳定。
- 提供记忆管理页面（查看、编辑、删除、锁定、忽略）。

验收标准：

- 同一用户多轮对话后，偏好命中率明显提升。
- 错误提取可人工纠正且不会重复污染。

## Phase 3：画像与长期学习（2 周）

- 基于原始记忆自动整理用户画像（身份、兴趣、表达风格、目标）。
- 建立“记忆衰减 + 强化”机制（高频被验证信息权重提升）。
- 引入反馈信号：用户评分、编辑、复制、追问作为学习依据。

验收标准：

- AI 回答更稳定地体现用户偏好，减少重复追问。
- 画像能解释“为什么给出这个回答”（可审计）。

## Phase 4：训练能力（持续演进）

- 样本集管道：对话 -> 清洗 -> 标注 -> 训练集版本化。
- 训练策略：先 LoRA/蒸馏，再按业务效果评估迭代。
- 加入 A/B 实验框架，比较“基础模型 vs 记忆增强 vs 训练增强”。

验收标准：

- 关键指标持续提升（满意度、任务完成率、复访对话时长）。

---

## 5. 关键技术建议

- 记忆分层：`短期上下文`、`长期事实`、`画像摘要` 分开存储与注入。
- 记忆检索：优先按语义相关度 + 时间衰减 + 重要性综合排序。
- 隐私治理：日志脱敏、敏感字段加密、用户级导出/删除能力。
- 观测体系：记录命中率、误提率、冲突率、注入 token 成本、延迟。

---

## 6. 风险与规避

- **风险：记忆污染**（错误提取导致越聊越偏）  
  规避：引入置信度阈值 + 人工可修订 + 冲突回滚。

- **风险：成本失控**（全量注入导致 token 暴涨）  
  规避：相关性召回 + 注入上限 + 摘要缓存。

- **风险：隐私合规问题**  
  规避：敏感信息分级存储、最小化日志、用户可删除与可追溯。

---

## 7. 下一步执行建议（可直接开工）

1. 先做 Phase 1（安全+真实流式+可取消），这是所有后续能力的地基。  
2. 并行推进 Phase 2 的记忆模型升级与管理页，确保“可纠错”。  
3. 上线一版小流量灰度，先验证记忆命中率和误提率。  
4. 指标稳定后再推进训练闭环，避免“坏数据训练坏模型”。

---

## 8. 成功标准（产品视角）

当用户在多天后再次进入对话，AI 能自然记住并使用以下信息，即可视为阶段成功：

- 用户长期偏好（喜欢/不喜欢、风格、语气）。
- 用户长期目标（学习、工作、计划）。
- 用户稳定习惯（时间偏好、表达方式、常用任务）。
- 用户显式反馈（“这次回答好/不好”）能影响后续行为。

---

## 9. 后端第一版（V1）落地范围

本次先做“不改动数据库结构的后端第一版”，基于现有 `user_memories` 数据直接提升效果与稳定性。

V1 目标：

- 在不增加训练成本的前提下，让小模型在有限上下文里“更像懂用户”。
- 避免把全部历史记忆硬塞进 prompt，减少噪声与 token 浪费。

V1 改动点（后端）：

1. **相关记忆召回注入**  
   - 从 `GetUserMemories` 返回集合中按“最近性 + 关键词命中”打分。  
   - 仅注入 TopN（如 8 条）记忆，替代“全量注入”。

2. **注入上限控制**  
   - 对注入条数和总字符数设置硬上限，避免挤占对话上下文预算。  
   - 当无明显命中时，回退注入少量最近记忆，保证鲁棒性。

3. **记忆去重与清洗**  
   - 对重复 key/value、空值、噪声值进行过滤。  
   - 避免 system prompt 中出现重复语义。

4. **敏感日志收敛**  
   - 移除“原始记忆提取内容”直接打印，改为计数与长度日志。  
   - 降低隐私泄露风险。

V1 非目标（后续 Phase 2 处理）：

- 不做表结构升级（`confidence/source/feedback` 等先不入库）。
- 不做训练/微调流水线。
- 不做跨设备全局记忆合并策略。

V1 验收标准：

- 同样的上下文窗口下，回答对用户偏好命中率提升。  
- Prompt token 消耗显著低于全量注入。  
- 日志中不再出现原始隐私记忆文本。
