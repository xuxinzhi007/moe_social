# 登录 / 注册 / Feed 统一性能优化说明

## 目标

- 优化登录与注册页邮箱候选输入时的重建开销。
- 优化 Feed 系列页面点赞流程，避免整页无意义重建。
- 为全局点赞状态缓存增加可回收能力，降低长时使用内存增长。

- `lib/pages/auth/login_page.dart`
- `lib/pages/auth/register_page.dart`
- `lib/widgets/email_completion_bubble.dart`
- `lib/pages/feed/home_page.dart`

## 核心优化点

### 1) 登录/注册：邮箱候选局部更新 + 输入防抖

- 将邮箱候选状态从整页 `setState` 迁移为 `ValueNotifier<List<String>>`。
- 使用 `ValueListenableBuilder` 仅重建候选气泡区域。
- 增加 `100ms` 防抖（`Timer`），减少高频输入期间候选计算次数。
- 焦点失去时取消防抖任务，避免无效回调。

收益：

- 降低输入触发的无关组件重建。
- 减少快速输入时抖动和短时 CPU 峰值。

### 2) Feed 页面点赞：统一改为局部刷新，不再整页 setState

- `HomePage` / `TopicPostsPage` / `UserProfilePage` 的 `onLike` 全部改为“仅同步帖子快照”，不再调用整页 `setState`。
- 点赞 UI 刷新仍由 `PostCard` 内部 `ValueListenableBuilder` 驱动。
- 保持页面本地帖子快照与最新点赞状态一致，避免后续重建回退旧值。
- 删除帖子时统一调用 `LikeStateManager.evictPost(...)`，避免残留缓存。

收益：

- 点赞高频交互不再触发页面级 rebuild。
- 滚动中点赞更平滑，降低掉帧风险。

### 3) LikeStateManager：新增缓存回收策略

- 新增帖子与评论点赞缓存的“触达时间戳”记录。
- 新增如下能力：
  - `evictPost / evictPosts`
  - `evictComment / evictComments`
  - `trimPostCaches`
  - `trimCommentCaches`
  - `trimCaches`
  - `debugStats`
- `PostService.getPosts` / `getComments` 统一负责 `sync + trim`：
  - `trimPostCaches(maxEntries: 600)`
  - `trimCommentCaches(maxEntries: 1200)`
- 避免“页面各自裁剪策略不一致”，统一收口到服务层。

收益：

- 长时间浏览大量帖子后，点赞缓存不会无界增长。
- 降低长期运行的内存压力和 OOM 风险。

## 交互一致性与可访问性补强

- 关键点击区域由 `GestureDetector` 替换为 `InkWell`（带水波纹）。
- 邮箱候选项增加 `Semantics` label，提升读屏可访问性。

## 后续建议

- 将 `LikeStateManager` 的回收触发从“页面内调用”升级为“全局调度策略”（例如应用进入后台时统一收缩）。
- 将首页评论数同步也改为局部更新（与点赞同策略）。
- 对 `FadeInUp` 在长列表场景加节流（如仅前 N 项动画），继续降低列表初次构建成本。
