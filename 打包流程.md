# Flutter 打包安卓安装包（APK）步骤
## 一、前置条件检查
在开始打包前，请确保电脑已配置好以下环境：
1.  Flutter SDK：已安装（当前版本 3.38.5）
2.  Android SDK：已安装
3.  JDK：已安装（Flutter 推荐使用 JDK 11 或 17）
4.  接受安卓许可：**当前安卓许可尚未完全接受，需手动操作**

## 二、打包流程步骤
### 第一步：接受安卓许可（必须手动操作）
该步骤需要在终端输入 `y` 同意相关条款，具体命令如下：
```bash
flutter doctor --android-licenses
```
运行后，系统会依次提示多项协议，**一路输入 `y` 并回车**，直到操作结束。

### 第二步：获取项目依赖
确保项目中所有插件都已下载完成，执行命令：
```bash
flutter pub get
```

### 第三步：执行打包命令
可根据需求选择**调试版**（测试用）或**发布版**（正式安装用）进行打包。

1.  **打包发布版（Release APK）—— 推荐用于安装**
    ```bash
    flutter build apk --release
    ```
2.  **打包调试版（Debug APK）—— 打包速度较快**
    ```bash
    flutter build apk --debug
    ```

## 三、预打包操作说明
1.  先执行 `flutter pub get` 进行项目依赖获取和基础检查
2.  若因**安卓许可（licenses）** 问题导致操作中断，请优先完成第一步的许可接受操作
3.  依赖获取完成后，可直接在终端运行发布版打包命令

## 四、打包结果路径
打包成功后，生成的 APK 文件默认路径为：
```
build\app\outputs\flutter-apk\app-release.apk
```

## 五、签名重要提示
1.  上述打包方式默认使用 Flutter 调试签名，适用于**个人使用**或**发给朋友测试安装**。
2.  若需将 APK 发布到应用商店（如华为应用市场、小米应用商店、Google Play 等），需额外配置正式签名：
    - 创建 `key.jks` 签名文件
    - 在项目的 `android/key.properties` 文件中配置签名信息
