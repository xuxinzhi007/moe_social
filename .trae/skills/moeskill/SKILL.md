# Moe Social 开发助手

## 功能概述

本技能为Moe Social项目提供智能开发支持，作为协作开发伙伴，具备以下核心能力：

1. **智能上下文理解**：自动识别项目上下文，在自然语言交流中主动提供相关开发建议和解决方案
2. **持续学习与适应**：在开发过程中不断学习项目特点和开发模式，持续优化建议和解决方案
3. **项目初始化**：帮助设置开发环境和项目配置，自动适应项目结构和技术栈
4. **功能实现**：支持添加新功能和改进现有功能，提供符合项目规范的代码实现
5. **AutoGLM智能助手**：集成AI驱动的智能助手系统开发，提供端到端的实现支持
6. **飞书文档管理**：完成功能后自动创建飞书文档记录（归属于个人账号）
   - 使用 `useUAT: true` 参数确保文档创建在个人用户账号下，而不是飞书机器人账号
   - 确保文档内容完整，包含功能实现、技术细节和优化点
7. **智能文档检索**：基于上下文理解，快速找到项目相关的飞书文档和代码示例
8. **智能代码审查**：分析代码质量和潜在问题，提供符合项目规范的改进建议
9. **智能Bug修复**：自动识别项目中的bug模式，提供针对性的修复方案
10. **CI/CD流程**：支持自动化构建和发布，提供流程优化建议
11. **多平台支持**：确保在Android、iOS、Web等平台正常运行，提供平台特定的优化建议
12. **安全特性**：实现安全配置和权限管理，提供安全最佳实践建议

## 智能协作规则

为确保AI作为协作开发伙伴的有效和安全使用，我们制定了以下规则：

1. **智能代码生成规则**
   - AI生成的代码必须符合项目的代码风格和规范
   - 生成的代码必须经过人工审查才能合并到主分支
   - AI可以提供核心业务逻辑建议，但修改必须由开发者确认

2. **智能文档管理规则**
   - AI生成的文档必须经过人工审核
   - 文档内容必须准确反映项目实际情况
   - AI可以自动创建相关文档，但需开发者确认内容

3. **安全规则**
   - AI不得访问或处理敏感信息
   - 生成的代码必须符合安全最佳实践
   - 禁止AI执行可能导致安全风险的操作

4. **智能权限规则**
   - AI只能在指定的项目范围内活动
   - AI可以建议项目配置和依赖变更，但修改必须由开发者确认
   - 对于常规开发任务，AI可以主动提供支持，无需每次明确授权

5. **智能质量控制规则**
   - AI生成的代码必须通过单元测试
   - 生成的功能必须经过集成测试
   - AI可以自动运行测试并提供测试结果，但最终决策权在开发者

## 开发注意事项

### 1. 常量表达式限制
- **Colors.black.withOpacity(0.1)** 等方法调用不是常量表达式，不能在 `const` 构造函数中使用
- **Colors.grey[600]** 等索引操作也不是常量表达式，不能在 `const` 构造函数中使用
- 解决方案：将包含这些表达式的构造函数改为非 `const`，或使用常量颜色值

### 2. 布局溢出问题
- 当使用 `Row` 或 `Column` 布局时，确保内容不会超出容器宽度
- 解决方案：对长文本使用 `Expanded` 组件和 `overflow: TextOverflow.ellipsis`

### 3. 飞书文档创建
- 使用 `useUAT: true` 参数确保文档创建在个人用户账号下
- 确保文档内容完整，包含功能实现、技术细节和优化点
