# Git Worktree 开发助手

## 角色定位
为 Moe Social 项目提供 Git Worktree 分支管理支持，简化功能开发和 bug 修复流程。

## 核心能力
- **创建开发分支**：基于当前分支创建新的功能/修复分支（命名格式：feature/ai-<原分支名>-<功能名> 或 fix/ai-<原分支名>-<问题描述>）
- **分支开发支持**：在独立 worktree 中进行开发，不影响主分支
- **代码合并**：开发完成后合并到原分支（根据分支名称识别目标分支）
- **分支清理**：删除临时开发分支

## 触发场景
当用户提到以下内容时触发：
- 开发新功能
- 修复 bug
- 创建分支
- 合并代码
- 清理分支

## 工作流程

### 1. 创建开发分支
```bash
# 根据用户需求创建分支
git checkout -b feature/ai-<功能名>
# 或
git checkout -b fix/ai-<问题描述>
```

### 2. 开发阶段
在新分支上进行代码修改和测试。

### 3. 合并确认
完成开发后，提示用户确认合并：
- 显示分支差异
- 询问用户是否确认合并
- 用户确认后执行合并

### 4. 清理分支
合并成功后，删除临时开发分支：
```bash
git branch -d feature/ai-<功能名>
```

## 分支命名规范
- **功能开发**: `feature/ai-<功能名>`
- **Bug 修复**: `fix/ai-<问题描述>`

## 使用示例
- "帮我开发一个新功能：用户头像上传" → 创建 feature/ai-user-avatar-upload
- "修复登录页面的 bug" → 创建 fix/ai-login-page-bug
- "合并当前分支到 main" → 确认后合并
- "删除临时分支" → 删除已完成的 ai 分支

## 注意事项
- 创建分支前会检查当前工作目录是否干净
- 合并前会提示用户确认
- 删除分支前会确认分支已合并

## 典型场景示例
1. **新功能开发**：用户需要开发新功能，自动创建 feature/ai-xxx 分支
2. **Bug 修复**：用户报告问题，创建 fix/ai-xxx 分支进行修复
3. **代码审查**：在独立分支中进行审查，确认后合并
4. **临时测试**：创建临时分支进行测试，完成后清理
